## Context

The `Relation` surface holds the single largest cluster (71 misses on
`relation.ts`, the union of three sub-groups), all Ruby metaprogramming
artifacts. This story owns the whole relation file group so the shared surface
isn't split across colliding PRs:

- `packages/activerecord/src/relation/query-methods.ts` (145/187, 42 miss) and
  `relation.ts` (328/399): the generated value accessors —
  `includes_values`/`includes_values=`, `eager_load_values`, `preload_values`,
  `joins_values`, `left_outer_joins_values`, `references_values`,
  `extending_values`, `unscope_values`, `optimizer_hints_values`,
  `annotate_values`, `with_values`, `readonly_value`, `reordering_value`,
  `strict_loading_value`, `reverse_order_value`, `create_with_value`,
  `skip_query_cache_value`, `where_clause`/`=`, `having_clause`/`=`,
  `from_clause`/`=`, `extensions`, `build_having_clause`. Generated by
  `Relation::VALUE_METHODS.each` (`query_methods.rb:162`); trails stores this
  state in private fields (`_whereClause`, `_referencesValues`) and
  `WhereClause`/`FromClause` objects.
- `relation/delegation.ts` (18/40, 22 miss) + `relation.ts`: Array/Enumerable
  delegations `to_xml, each, join, sample, reverse, rotate, compact, in_groups,
in_groups_of, to_sentence, to_fs, to_formatted_s, as_json, shuffle, split,
slice, index, rindex` (`delegation.rb:101` `delegate … to: :records`) and klass
  delegations `primary_key, with_connection, connection, table_name,
transaction, sanitize_sql_like, name, klass` (`delegation.rb:106`
  `to: :model`). trails realizes these via a runtime delegation list, not named
  methods.
- Stragglers: `relation/finder-methods.ts` `member?`, `relation/where-clause.ts`
  `any?`, `relation/calculations.ts` `build_count_subquery`, `relation.ts`
  `perform_calculation`.

## Acceptance criteria

- Each generated value accessor and delegation is either ported as a thin
  named accessor/delegator OR added to a `SKIP_GROUPS` entry (or
  `unported-files` exclusion) in `scripts/api-compare/conventions.ts` with a
  reason citing the private-field/runtime-delegation realization. Read each
  against the Rails source first — if any accessor gates behavior trails can't
  reach by another name, port it.
- `member?`, `any?`, `build_count_subquery`, `perform_calculation` ported or
  skip-listed with reason.
- `pnpm api:compare --package activerecord` shows relation.ts,
  relation/query-methods.ts, relation/delegation.ts, relation/finder-methods.ts,
  relation/where-clause.ts, relation/calculations.ts at 100%.
- PR stays under 500 LOC; if genuine ports push it over, register the overflow
  as a follow-up story rather than splitting the file group.
