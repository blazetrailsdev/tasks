## Context

The Ruby API extractor already records, per method, the set of method names
its body invokes â€” `MethodInfo.calls` (see `scripts/api-compare/types.ts`,
populated by `scripts/api-compare/extract-ruby-api.rb`). This data is currently
unused by any lint rule. A spike confirmed the calls arrays contain
fidelity-critical lifecycle primitives in the `activerecord` package:
`run_callbacks` appears 11Ă—, `transaction` 26Ă—, `with_lock`, and
`clear_attribute_changes`. (Note: `super` is NOT captured by the Ripper
extractor â€” count 0 â€” so a "must call super" rule is out of scope here.)

A general "the ported TS method must make every call Rails makes" rule is a
noise trap: TS legitimately restructures bodies (extracted helpers, inlined
calls, idiom swaps), so it would produce overwhelming false positives. This
story scopes to the high-signal subset only: **callback invocation**.

The repo already has the proven pattern this should follow â€” a generated JSON
manifest consumed by a custom ESLint rule. See the existing rules and their
baselines: `eslint/rails-private-methods.mjs` + `rails-private-methods.json`,
`eslint/rails-deprecated-methods.json`, `eslint/rails-file-structure-method-order.mjs`.
These JSON artifacts are generated by `compare.ts` / the manifest builders
invoked from `scripts/api-compare/orchestrate.ts`.

Rails source to mirror: methods whose body calls `run_callbacks(:<event>)`
(e.g. `ActiveRecord::Persistence#save`, transaction/touch paths). The TS
equivalent is `runCallbacks("<event>")`.

## Acceptance criteria

- A generated JSON artifact (e.g. `eslint/rails-callback-invocations.json`)
  mapping Rails method name â†’ required callback events, derived from the
  `run_callbacks` entries in `MethodInfo.calls`. Generated in the same place
  the other rails-\*.json baselines are produced (orchestrate/compare path),
  so `pnpm api:compare` refreshes it.
- A custom ESLint rule (`eslint/rails-callback-invocations.mjs`) that, for a
  ported TS method whose name matches a Rails method with required callback
  events, flags the method if its body does not call `runCallbacks("<event>")`
  for each required event. Scope the rule to `packages/activerecord/src/**`.
- An exclude baseline JSON (mirroring `*-exclude.json` convention) so the rule
  ratchets â€” existing violations are grandfathered and the list only shrinks.
- A `.test.mjs` for the rule covering: matching method missing the call (flag),
  matching method with the call (pass), excluded file (pass), non-matching
  method (pass).
- Rule is wired into the ESLint config like the sibling rails-\* rules.

## Out of scope

- General missing-call parity (the noise trap above).
- `super`-invocation rules (not captured by the extractor).
- The symmetric TS-side `calls` capture + call-set parity advisory dimension in
  compare.ts â€” that is a separate, concurrent piece of work (Family B).
